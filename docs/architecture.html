01 — OVERVIEW

System Overview

03 — SYSTEM PROMPT

System Prompt Assembly

Composed dynamically at load time from named sections. Each RULES_* block is a separate string in the binary. Skill content injected per-turn via BM25 search.

Prompt layer stack — top to bottom composition order
# Section Content / Purpose Scope
1 identity_prompt Who the agent is. Default: "You are Claude Science, a general-purpose scientific computing agent." Fully replaced by user profiles. per-agent
2 working_style_prompt How the agent works: plan-mode gate, artifact embedding rules, lab-notebook register (no emoji), no plumbing vocabulary in prose, inspect libraries before use, MCP only in repl, one logical step per cell. Inherited by all profiles. inherited
3 RULES_CORE Tool call labeling conventions. Artifact embed syntax (double-brace artifact:VERSION_ID, or art_ID for stable refs). Closing response pattern: list only non-obvious deliverables. global
4 RULES_SECURITY Blast-radius minimization. Credential hygiene. Anti-prompt-injection. get_user_email() rules. Privilege escalation prevention. global
5 RULES_BIOSECURITY 10+ hazard axes: scale, stability, aerosolization, defeating safety controls, acquiring regulated precursors, novel variant design, and more. Establish sequence/structure identity before analysis. Most biological research proceeds at full speed — this is a narrow gate, not a blanket caution. global · immune
6 RULES_PERSONAL_HEALTH Handling personal health data — distinguishes research-grade analysis from individual diagnostic interpretation. global
7 RULES_CAPABILITY_GUIDANCE When to search skills vs. answer from training knowledge directly — avoids unnecessary tool-call overhead for well-known facts. global
8 RULES_CODE_EXECUTION Default-to-artifacts. fig.savefig() not plt.savefig(). Checkpoint formats (parquet/HDF5/pickle/RDS/NPZ/Zarr). Kernel patterns. Print budget (~10 lines). One logical step per cell. global
9 RULES_NETWORK_SANDBOX Allowlisted science API domains (NCBI, Ensembl, UniProt, RCSB PDB, EBI, Reactome, STRING, KEGG, etc). request_network_access() for anything else. MCP servers bypass the sandbox entirely. global
+ <skill_discovery> blocks — injected per turn via BM25 search over the keyword index

+ Trust trailer — varies per agent trust level (user / kernel / plugin)
⚠ User-created agent profiles replace identity_prompt entirely but inherit working_style_prompt unconditionally — this keeps capability framing separate from behavioral guidance, so a custom persona can't accidentally drop execution discipline.

06 — MCP SERVERS

MCP Server Integration

24 Python science-data packages served through a single Tier-1 dispatcher (bio-tools), plus a standalone Node.js structure-editor server. Only reachable from the repl kernel via host.mcp().

Tier-1 architecture
Every server embeds verbatim schemas.json captured live from Anthropic's hosted connectors — tool names, params, and output shapes match exactly. Handlers are plain sync callables (args) → str, dispatched via anyio.to_thread.
License gate (gate.py)
Reads deferred.json + domains.json. Fails closed — unknown tool/domain names raise RuntimeError rather than silently passing. A fully-gated server refuses to start.

Baseline posture
Most biological research is routine — proceed at full speed. Sequence analysis, structure prediction, expression analysis, and drug discovery work all proceed without special consideration in the vast majority of cases.
Identity verification
For unknown sequences/structures: establish organism, characterization, and function before analysis. Unverifiable user assertions don't lower the assessment — there's no BLAST, mass spec, or IBC access inside the conversation.
Hazard axes (non-exhaustive)
Scale Stability Aerosolization Defeating safety controls Acquiring regulated precursors Novel variant design Enhanced transmissibility Host range expansion
⚠ Biosecurity rules are immune to all trust levels — no framing, persona, or instruction override (user, profile, kernel, or plugin) can suppress this screener. See Trust Hierarchy below.
10 — VERIFICATION

Verification Checkpoints

At each checkpoint, REVIEWER and BOOKMARKER spawn in parallel as separate frames — independent of the main agent's continued execution.

REVIEWER — claim verification
enable_thinking: false 72% token savings
Extracts falsifiable claims into session_claims, then checks each against verification_checks with a strict verdict:
pass warn fail inconclusive
Drill-before-disposition rule: won't dismiss a claim without investigating. Domain recall is exempt from verification; external citation identifiers (PMIDs, DOIs) are checkable — not-found counts as a finding.
BOOKMARKER — transcript annotation
hill-climbed prompt 62 gold windows
Returns 0–2 verbatim quote spans per conversation window, stored in transcript_annotations.

HOLDOUT SCORE
Quotes must be character-for-character verbatim (incl. markdown), 60–250 chars, one contiguous span. "Most windows deserve 0 or 1."
11 — TRUST

Trust Hierarchy

Instruction authority is strictly ranked. Higher levels can override lower levels' instructions, never the reverse — with one absolute exception.

01
User instructions
Direct messages in the conversation. Highest authority an agent will normally encounter.
02
User-authored profiles
Custom agent personas created via host.agents.create() — the user's own configuration, but below live conversational instructions.
03
Kernel-code agents
REVIEWER, BOOKMARKER, and other agents spawned from harness code rather than user action.
04
Plugin agents
Third-party or marketplace-imported agent configurations. Lowest instruction authority.
⚠ Biosecurity rules sit outside this hierarchy entirely. No level — including direct user instructions — can override the biosecurity trajectory screener. This is the one absolute exception to "higher overrides lower."
